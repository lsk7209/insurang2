# 수동 배포 가이드

## GitHub 배포 방법

### 1. 변경사항 확인
```bash
git status
```

### 2. 변경사항 스테이징
```bash
# 모든 변경사항 추가
git add .

# 또는 특정 파일만 추가
git add app/admin/offers/page.tsx
```

### 3. 커밋
```bash
git commit -m "관리자 페이지 개발 완료"
```

### 4. GitHub에 푸시
```bash
# main 브랜치에 푸시
git push origin main

# 또는 master 브랜치인 경우
git push origin master
```

### 5. Cloudflare Pages 자동 배포 확인
- GitHub에 푸시하면 Cloudflare Pages가 자동으로 빌드를 시작합니다
- Cloudflare 대시보드에서 배포 상태를 확인할 수 있습니다

## 빌드 오류가 있는 경우

현재 빌드 오류가 있지만 강제로 배포하려면:

### 옵션 1: 빌드 오류 무시하고 푸시 (권장하지 않음)
```bash
git add .
git commit -m "WIP: 관리자 페이지 개발 중"
git push origin main
```
⚠️ **주의**: 빌드 오류가 있으면 Cloudflare Pages 배포가 실패합니다.

### 옵션 2: 빌드 오류 수정 후 배포 (권장)
1. 빌드 오류를 먼저 수정
2. `npm run build`로 빌드 성공 확인
3. 그 다음 GitHub에 푸시

## Cloudflare Pages 수동 배포

### Wrangler CLI 사용
```bash
# Wrangler 설치 (없는 경우)
npm install -g wrangler

# 로그인
wrangler login

# 빌드
npm run build

# 배포
npx wrangler pages deploy .next --project-name=your-project-name
```

## 현재 상태 확인

### Git 상태 확인
```bash
git status
```

### 원격 저장소 확인
```bash
git remote -v
```

### 브랜치 확인
```bash
git branch
```

## 빠른 배포 명령어 (한 줄씩 실행)

```bash
# 1. 변경사항 확인
git status

# 2. 모든 변경사항 추가
git add .

# 3. 커밋
git commit -m "관리자 페이지 개발 완료"

# 4. 푸시
git push origin main
```

## 문제 해결

### 푸시가 거부되는 경우
```bash
# 원격 저장소의 최신 변경사항 가져오기
git pull origin main

# 충돌 해결 후 다시 푸시
git push origin main
```

### 커밋 메시지 수정
```bash
git commit --amend -m "새로운 커밋 메시지"
git push origin main --force
```

⚠️ **주의**: `--force` 옵션은 신중하게 사용하세요.
